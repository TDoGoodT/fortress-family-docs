# DevSkill — ניתוח קוד ותכנון פיצ׳רים

## תיאור
מאפשר למנהל המערכת לנתח את מבנה הקוד של פורטרס, לשאול שאלות על הארכיטקטורה, ולתכנן פיצ׳רים חדשים.

## פקודות

| תבנית | פעולה | דוגמה |
|--------|-------|-------|
| `index` / `אנדקס` | בניית אינדקס קוד | אנדקס |
| `dev query <שאלה>` | שאילתה על מבנה הקוד | dev query מה ה-skills שיש? |
| `dev plan <תיאור>` | תכנון פיצ׳ר חדש | dev plan הוסף skill לניהול תקציב |

## הרשאות
- מנהל בלבד (`is_admin == True`)
- משתמשים רגילים מקבלים הודעת "אין הרשאה"

## תיוג מקורות (Source Attribution)
כל תשובה חייבת לתייג כל טענה עם מקור:

| תגית | משמעות |
|-------|--------|
| `indexed_fact` | עובדה ישירה מהאינדקס — קובץ קיים, מחלקה נמצאה, import מפורש |
| `inferred_pattern` | דפוס שנצפה מניתוח חוצה-רכיבים — "כל ה-skills יורשים מ-BaseSkill" |
| `llm_assumption` | הנחה שאינה מבוססת על ראיות מהקוד — ידע כללי של ה-LLM |

## הנחיות לתשובות
- ענה בעברית כשהשאלה בעברית
- היה תמציתי — WhatsApp הוא ערוץ סיכום, הקובץ הוא מקור האמת
- אם התשובה ארוכה מ-500 תווים, שמור קובץ Markdown מלא ושלח סיכום
- אל תחשוף נתיבי קבצים, שמות פונקציות או מבנה קוד למשתמשים שאינם מנהלים
